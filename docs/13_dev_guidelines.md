# 開発ガイド

## 目的
- 実装の一貫性と保守性を高める

## コーディング規約
- Python 3.10+ 対応を前提
- すべての侧面効果はインフラレイヤに寄せる
- 入力バリデーションはユースケースで行う
- 例外はログに残して UI を止めない

## ディレクトリ運用
- UI / UseCase / Domain / Infra を分離
- テンプレトは templates/ に集中
- 実行時成果物（DB/ログ）は data/ と logs/

## スレッド方針
- UI スレッドで HTTP をしない
- ワーカーで取得し、signal で反映

## ログ方針
- INFO/WARN/ERROR で分類
- ネットワーク例外は必ず保存

## テスト
- ドメインの純粋ロジックはユニットテスト
- UI は手動テスト主体
